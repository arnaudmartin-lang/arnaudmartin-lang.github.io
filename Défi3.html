Défi 3 : Répertoires

Pour trouver ce red flag, nous nous sommes aidés de l’indice. Nous avons trouvé un code que nous avons décrypté, ce qui nous a donné un message. Ensuite, nous avons ajouté un mot dans l’URL du site qui nous a menés à un endroit caché.

Red flag = *Dossiers_non_securises_007

Pour éviter le vol de données personnelles, il faudrait que seul l’admin ait accès aux configurations du site ou sécurise les répertoires sensibles (ex : .htaccess).

Risques professionnels :
- Fuite de données sensibles et confidentielles
- Exposition des données clients
- Accès à des fichiers internes (documents, employés, sauvegardes)
