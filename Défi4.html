Défi 4 : URL ?

Pour trouver ce red flag, nous avons modifié l’URL pour accéder à une page contenant plusieurs mots de passe. Après avoir trouvé le bon mot de passe, nous avons modifié l’URL en “admin”, puis saisi le mot de passe pour obtenir le red flag.

Red flag = une_URL_ca_se_ch@nge!

Ce type de faille permet à un attaquant de modifier les paramètres d’une URL pour accéder à des pages non autorisées.

Risques professionnels :
- Fuite de données et perte de confidentialité
- Accès aux données clients (identité, informations sensibles)
- Accès à des bases internes ou pages d’administration
